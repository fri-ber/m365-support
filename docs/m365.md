# ☁️ Microsoft 365

Anleitungen für häufige Aufgaben im Microsoft 365 Admin Center und Entra ID.

---

## Shared Mailbox einrichten

Eine Shared Mailbox ermöglicht mehreren Benutzern das gemeinsame Lesen und Senden von E-Mails.

### Über das Admin Center

1. **Exchange Admin Center** → *Recipients* → *Mailboxes*
2. Auf **+ Add a shared mailbox** klicken
3. Name und E-Mail-Adresse eingeben → *Save*
4. Unter **Mailbox delegation** die Berechtigungen vergeben:
   - `Full Access` — Postfach lesen & verwalten
   - `Send As` — E-Mails im Namen der Mailbox senden
   - `Send on Behalf` — Senden mit Hinweis auf Stellvertretung

### Per PowerShell

```powershell
# Shared Mailbox erstellen
New-Mailbox -Shared -Name "Support" -DisplayName "IT Support" `
  -Alias "support" -PrimarySmtpAddress "support@firma.ch"

# Full Access vergeben
Add-MailboxPermission -Identity "support@firma.ch" `
  -User "max.muster@firma.ch" -AccessRights FullAccess -AutoMapping $true

# Send As vergeben
Add-RecipientPermission -Identity "support@firma.ch" `
  -Trustee "max.muster@firma.ch" -AccessRights SendAs -Confirm:$false
```

> **Tipp:** AutoMapping `$true` hängt die Mailbox automatisch in Outlook ein, ohne dass der User etwas konfigurieren muss.

## MFA aktivieren (Entra ID)

### Conditional Access Policy (empfohlen)

1. **Entra Admin Center** → *Security* → *Conditional Access* → *New Policy*
2. Name: `Require MFA – All Users`
3. **Users**: All users (Ausnahmen für Break-Glass-Accounts!)
4. **Cloud apps**: All cloud apps
5. **Grant**: ✅ Require multifactor authentication
6. Policy auf **On** stellen → *Save*

### Per-User MFA (Legacy)

> ⚠️ Microsoft empfiehlt Conditional Access statt Per-User MFA.

1. **Microsoft 365 Admin Center** → *Users* → *Active users*
2. Oben **Multi-factor authentication** auswählen
3. User markieren → *Enable* oder *Enforce*

### MFA für einen User zurücksetzen

```powershell
# MFA-Methoden zurücksetzen via Graph
Connect-MgGraph -Scopes "UserAuthenticationMethod.ReadWrite.All"

$userId = (Get-MgUser -UserId "user@firma.ch").Id
Get-MgUserAuthenticationMethod -UserId $userId
```

## Lizenzen verwalten

### Einzelne Lizenz zuweisen

1. **Entra Admin Center** → *Users* → User auswählen
2. *Licenses* → *+ Assignments*
3. Lizenz auswählen, ggf. einzelne Apps deaktivieren → *Save*

### Bulk-Zuweisung per PowerShell

```powershell
Connect-MgGraph -Scopes "User.ReadWrite.All", "Directory.ReadWrite.All"

# Verfügbare SKUs anzeigen
Get-MgSubscribedSku | Select-Object SkuPartNumber, SkuId, ConsumedUnits

# Lizenz zuweisen
$licenseId = "<SKU-ID hier>"
$addLicenses = @(@{ SkuId = $licenseId })

Set-MgUserLicense -UserId "user@firma.ch" `
  -AddLicenses $addLicenses -RemoveLicenses @()

Write-Host "✅ Lizenz zugewiesen" -ForegroundColor Green
```

### Alle lizenzierten User exportieren

```powershell
Get-MgUser -All -Filter "assignedLicenses/`$count ne 0" `
  -ConsistencyLevel eventual -CountVariable count |
  Select-Object DisplayName, UserPrincipalName, AccountEnabled |
  Export-Csv "./Lizenzen-Report.csv" -NoTypeInformation -Encoding UTF8
```

## SharePoint Berechtigungen

### Site-Berechtigungen anpassen

1. SharePoint Site öffnen → ⚙️ *Settings* → *Site permissions*
2. **Advanced permissions settings** klicken
3. Gruppen & Berechtigungsstufen anpassen:
   - `Full Control` — Vollzugriff
   - `Edit` — Inhalte bearbeiten
   - `Read` — Nur lesen

### Vererbung aufheben

Für einzelne Bibliotheken oder Ordner:
1. Bibliothek öffnen → ⚙️ → *Library settings* → *Permissions for this document library*
2. *Stop Inheriting Permissions* → eigene Berechtigungen setzen

> ⚠️ Aufgebrochene Vererbungsketten sind schwer zu verwalten — sparsam einsetzen!

## Teams: Gruppen & Gäste

### Neue Team-Gruppe erstellen

1. **Teams** → *Teams* → *Join or create a team*
2. **Create team** → Typ wählen:
   - `Private` — nur auf Einladung
   - `Public` — jeder in der Organisation kann beitreten
3. Mitglieder hinzufügen, Kanäle strukturieren

### Externe Gäste einladen

```powershell
# Gast per Graph API einladen
Connect-MgGraph -Scopes "User.Invite.All"

New-MgInvitation -InvitedUserEmailAddress "gast@extern.com" `
  -InviteRedirectUrl "https://teams.microsoft.com" `
  -SendInvitationMessage:$true
```
